HTML · Topic 1 of 4

HTML Document Structure

Every webpage starts as an HTML file. Think of it as the skeleton of a human body — it holds everything in place!

💡

Simple Idea: HTML is like the structure of a house. The walls, doors, and rooms are made of HTML. CSS paints the walls. JavaScript turns on the lights!

🏗️ The Basic HTML Template

Every HTML file looks like this. Copy this whenever you start a new project!

index.html
<!-- Line 1: Tell the browser this is HTML5 -->
<!DOCTYPE html>

<html lang="en">   <!-- The ROOT element that wraps EVERYTHING -->

  <head>               <!-- INVISIBLE part — info for the browser -->
    <meta charset="UTF-8">              <!-- Support all languages -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Webpage</title>     <!-- Tab title -->
    <link rel="stylesheet" href="style.css">  <!-- Link CSS -->
  </head>

  <body>               <!-- VISIBLE part — everything users see -->
    <h1>Hello World! 👋</h1>
    <p>This is my first webpage!</p>
    <script src="app.js"></script>         <!-- Link JS at the bottom -->
  </body>

</html>
🧠 <head> — The Brain

Invisible to users. Contains the page title, links to CSS files, and meta information for Google.

👀 <body> — The Face

Everything inside <body> is visible on the screen — text, images, buttons, everything!

🏷️ Opening & Closing Tags

Most tags need an opening <p> AND a closing </p>. The / means "end this".

📐 Nesting Elements

Elements live inside other elements like Russian dolls. Always close in reverse order!

📝 Common HTML Tags You Need to Know

Common Tags

<h1>Main Title (Only ONE h1 per page!)</h1>
<h2>Section Title</h2>
<h3>Sub Section Title</h3>

<!-- TEXT -->
<p>A paragraph of text. Most content goes in <p> tags.</p>
<strong>This is important (bold)!</strong>
<em>This is emphasized (italic).</em>
<br>  <!-- Line break — no closing tag needed -->
<hr>  <!-- Horizontal line -->

<!-- LINKS -->
<a href="https://google.com">Click me to go to Google!</a>
<a href="about.html" target="_blank">Opens in new tab</a>

<!-- LISTS -->
<ul>  <!-- Unordered list (bullets) -->
  <li>Apples 🍎</li>
  <li>Bananas 🍌</li>
</ul>

<!-- CONTAINERS -->

Semantic HTML Elements

Semantic means "having meaning". These tags tell browsers AND humans what the content is about!

🤔

Why Semantic? Imagine reading a book where every chapter is just called "section". Confusing! Semantic tags are like chapter names — they tell you what's inside: "Introduction", "Conclusion", etc.

🏠 A Webpage is Like a Newspaper

<header> Logo + Top Nav
<nav> Menu Links
<main> <article> Main Content Here
<footer> Copyright, Links
Semantic Layout
<header>
  <nav>
    <a href="/">Home</a>
    <a href="/about">About</a>
  </nav>
</header>

<main>
  <article>            <!-- A complete, standalone story -->
    <h2>My Blog Post</h2>
    <p>Content here...</p>
  </article>

  <section>            <!-- A group of related content -->
    <h2>Related Topics</h2>
  </section>

  <aside>              <!-- Sidebar content -->
    <p>Advertisement or tips</p>
  </aside>
</main>

<footer>
  <p>&copy; 2024 My Website</p>
</footer>
TagWhat It MeansWhere to Use
<header>Top section — logo, main navTop of page or top of an article
<nav>Navigation linksInside header, or standalone menu
<main>The main unique contentOnly ONE per page — the star!
<article>Self-contained piece of contentBlog post, news story, card
<section>Group of related contentChapter-like groups
<aside>Related but not main contentSidebar, tips box, ads
<footer>Bottom section — links, copyrightBottom of page or article
<figure>Image with captionWrap <img> + <figcaption>
<time>A date or time<time datetime="2024-01-15">Today</time>
⚠️

Don't use <div> for everything! Many beginners use <div> for everything. Use semantic tags first — they're better for Google search rankings and screen readers that help blind users.

🎨

Simple Idea: If HTML is the skeleton, CSS is the clothes and makeup. HTML says "this is a heading", CSS says "make that heading red, big, and centered".

CSS · Topic 2 of 7

Selectors & Specificity

Selectors are how CSS finds the right elements to style. Specificity decides which rule wins when two rules conflict!

🎯 Types of Selectors

CSS Selectors
/* ELEMENT selector — targets all <p> tags */
p { color: gray; }

/* CLASS selector — targets elements with class="card" */
.card { background: white; border-radius: 8px; }

/* ID selector — targets element with id="logo" (use ONCE only!) */
#logo { width: 120px; }

/* GROUP selector — style multiple at once */
h1, h2, h3 { font-family: 'Baloo 2', cursive; }

/* DESCENDANT — all <a> inside <nav> */
nav a { color: white; text-decoration: none; }

/* CHILD — only DIRECT children */
ul > li { list-style: none; }

/* PSEUDO-CLASS — :hover, :focus, :first-child */
button:hover  { background: darkblue; }
input:focus   { border-color: blue; }
li:first-child { font-weight: bold; }
li:nth-child(2n) { background: #f0f4ff; } /* every 2nd */

/* PSEUDO-ELEMENT — ::before, ::after (content tricks!) */
p::first-line  { font-weight: bold; }
.icon::before { content: '★ '; }

/* ATTRIBUTE selector */
input[type="email"] { border-color: green; }
a[href^="https"]    { color: green; } /* starts with https */

🏆 Specificity — Who Wins?

When two rules target the same element, the one with higher specificity wins. Think of it like a high score!

CSS · Topic 3 of 7

The Box Model

Every single HTML element is a rectangular box. Once you understand this, CSS layout becomes easy!

📦

Think of a gift box: The gift = content. Bubble wrap = padding. The box walls = border. Space between boxes on a shelf = margin!

Interactive Box Model VISUAL
MARGIN — outside space
BORDER — visible edge
PADDING — inner space
CONTENT
width × height
Box Model CSS
.card {
  /* CONTENT */
  width: 300px;
  height: 200px;

  /* PADDING — space inside the border */
  padding: 20px;                /* all 4 sides */
  padding: 10px 20px;           /* top/bottom  left/right */
  padding: 10px 20px 15px 5px; /* top right bottom left (clockwise) */
  padding-top: 20px;            /* individual sides */

  /* BORDER */
  border: 2px solid #e2e8f0;
  border-radius: 12px;          /* round corners */
  border-radius: 50%;           /* perfect circle! */

  /* MARGIN — space OUTSIDE the border */
  margin: 20px;
  margin: 0 auto;               /* center a block element */
  margin-bottom: 16px;

  /* IMPORTANT: Makes width include padding+border */
  box-sizing: border-box;
}

/* ALWAYS add this at the top of your CSS file! */
*, *::before, *::after {
  box-sizing: border-box;       /* prevents layout headaches */
}

✅ padding vs margin

  • Padding = space inside (background fills it)
  • Margin = space outside (transparent)
  • Use padding to grow a button. Use margin to push it away from others.

📏 box-sizing: border-box

By default, if you set width: 100px and add padding: 20px, the total width becomes 140px. With border-box, it stays 100px. Always use it!

Positioning & Display

Control exactly where elements appear on the page — in the flow, out of the flow, fixed in the corner!

📌 The 5 Position Values

Position Demo VISUAL
static (normal flow)
relative (offset 16px from normal)
absolute (bottom-right of parent)
fixed (top-right, stays on scroll)
CSS Positioning
/* ── STATIC (default) ──────────────────── */
.normal {
  position: static;  /* normal flow, top/left/etc IGNORED */
}

/* ── RELATIVE ────────────────────────────── */
.moved {
  position: relative;
  top: 20px;           /* move 20px DOWN from normal spot */
  left: 10px;          /* move 10px RIGHT from normal spot */
  /* Other elements still see it in the original spot */
}

/* ── ABSOLUTE ────────────────────────────── */
.parent { position: relative; }  /* parent must be positioned! */
.badge {
  position: absolute;
  top: 0; right: 0;   /* top-right corner of parent */
  /* Completely removed from normal flow */
}

/* ── FIXED ───────────────────────────────── */
.navbar {
  position: fixed;
  top: 0; left: 0; right: 0;
  /* Stays at top even when you scroll! */
}

/* ── STICKY ──────────────────────────────── */
.section-header {
  position: sticky;
  top: 0;            /* sticks when it reaches the top */
}

📺 Display Property

inlineFlows in text, can't set width/height<span>, <a>, <strong>

visibility: hidden vs display: none — display: none removes the element completely (no space). visibility: hidden makes it invisible but it still takes up space. Like Harry Potter's cloak vs. disappearing!

CSS · Topic 5 of 7

Flexbox & Grid Basics

The two modern layout systems that made CSS layout fun again. Flexbox = one direction. Grid = two directions!

🏋️ Flexbox — One Row or Column

💡

When to use Flexbox? Navigation bars, centering something on a page, cards in a row, aligning items. Any time you want things in a LINE!

Flexbox Playground INTERACTIVE
Box 1
Box 2
Box 3
Box 4
justify-content (horizontal)
align-items (vertical)
flex-direction
Flexbox CSS
/* PARENT — the flex container */
.container {
  display: flex;                     /* activate flexbox! */
  flex-direction: row;              /* row | column | row-reverse */
  justify-content: space-between;  /* alignment on MAIN axis */
  align-items: center;            /* alignment on CROSS axis */
  flex-wrap: wrap;               /* wrap to next line when full */
  gap: 1rem;                     /* space between items */
}

/* CHILDREN — the flex items */
.item {
  flex: 1;                       /* grow and shrink equally */
  flex-grow: 2;                  /* grow twice as much as others */
  flex-shrink: 0;               /* don't shrink */
  flex-basis: 200px;            /* starting size */
  align-self: flex-end;         /* override align-items for this item */
  order: 2;                     /* change visual order */
}

/* PERFECT CENTERING — the classic use case */
.center-me {
  display: flex;
  justify-content: center;
  align-items: center;
  /* The child is now perfectly centered! */
}

🔲 CSS Grid — Rows AND Columns

💡

When to use Grid? Page layouts, photo galleries, dashboards. Any time you want things in a TABLE-LIKE GRID with rows AND columns!

Grid Playground INTERACTIVE
Span 2 Columns
Box 2
Box 3
Box 4
Full Width Row
grid-template-columns
gap
CSS Grid
/* BASIC GRID */
.gallery {
  display: grid;
  grid-template-columns: repeat(3, 1fr);  /* 3 equal columns */
  grid-template-columns: 200px 1fr 1fr;  /* fixed + flexible */
  grid-template-rows: auto 1fr auto;
  gap: 1.5rem;
}

/* SPAN multiple columns/rows */
.wide-card { grid-column: span 2; }
.tall-card  { grid-row: span 2; }
.full-row   { grid-column: 1 / -1; }  /* from first to last */

/* NAMED AREAS (super readable!) */
.page {
  display: grid;
  grid-template-areas:
    "header header header"
    "nav    main   aside"
    "footer footer footer";
  grid-template-columns: 160px 1fr 200px;
  grid-template-rows: 60px 1fr 50px;
}
.header { grid-area: header; }
.main   { grid-area: main; }
.footer { grid-area: footer; }

/* RESPONSIVE grid (no media queries needed!) */
.auto-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  /* Auto-creates as many 250px-min columns as fit! */
}

Responsive Design Principles

Responsive websites look good on EVERY screen — phones, tablets, laptops, huge monitors. One code, all screens!

📱

Important fact: Over 60% of web traffic comes from mobile phones! If your site doesn't work on mobile, you're losing more than half your visitors!

📐 Viewport Meta Tag (Required!)

Required in <head>
<!-- Without this, mobile browsers zoom out and show tiny text! -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

Media Queries — Mobile First
/* ── MOBILE FIRST approach ────────────────────
   Start with mobile styles (no media query needed)
   Then ADD styles for bigger screens
──────────────────────────────────────────────── */

/* Default: Mobile (1 column) */
.grid {
  display: grid;
  grid-template-columns: 1fr;
  gap: 1rem;
}
.sidebar { display: none; }           /* hidden on mobile */

/* Tablet (576px and wider) */
@media (min-width: 576px) {
  .grid { grid-template-columns: repeat(2, 1fr); }
}

/* Laptop (768px and wider) */
@media (min-width: 768px) {
  .grid { grid-template-columns: repeat(3, 1fr); }
  .sidebar { display: block; }         /* show sidebar */
  .hamburger { display: none; }        /* hide mobile menu */
}

/* Desktop (1024px and wider) */
@media (min-width: 1024px) {
  .grid { grid-template-columns: repeat(4, 1fr); }
}

/* ── RESPONSIVE UNITS ──────────────────────── */
h1 { font-size: clamp(1.8rem, 5vw, 4rem); }  /* fluid! */
img { max-width: 100%; height: auto; }       /* responsive images */
.container { padding: 0 1rem; max-width: 1200px; margin: 0 auto; }

Modern CSS Workflows

Professional developers use these techniques to write clean, maintainable CSS. These are today's best practices!

🎨 CSS Custom Properties (Variables)

CSS Variables
/* ── DEFINE variables in :root (global) ───────── */
:root {
  --primary: #264de4;
  --secondary: #e8532a;
  --text: #2d2d44;
  --bg: #f0f4ff;
  --radius: 12px;
  --shadow: 0 4px 20px rgba(0,0,0,0.08);
  --font-body: 'Nunito', sans-serif;
}

/* ── USE variables with var() ──────────────────── */
.card {
  background: var(--bg);
  border: 2px solid var(--primary);
  border-radius: var(--radius);
  box-shadow: var(--shadow);
  color: var(--text);
}

/* Change one variable = change entire theme! */
.dark-mode {
  --bg: #1a1a2e;
  --text: #ffffff;
}

🏗️ BEM — Block Element Modifier

BEM is a naming convention that makes your CSS classes easy to understand and not conflict with each other.

BEM Naming
/* BEM = Block__Element--Modifier */

/* BLOCK — the component */
.card { background: white; border-radius: 12px; }

/* ELEMENT — a part of the block (uses __ double underscore) */
.card__title   { font-size: 1.2rem; font-weight: 700; }
.card__image   { width: 100%; }
.card__button  { padding: 0.5rem 1rem; }

/* MODIFIER — a variation (uses -- double dash) */
.card--featured     { border: 2px solid gold; }
.card--small        { max-width: 200px; }
.card__button--red  { background: red; }

/* HTML example */
<!--
<div class="card card--featured">
  <img class="card__image" ...>
  <h2 class="card__title">Title</h2>
  <button class="card__button card__button--red">Buy</button>
</div>
-->

/* ── OTHER MODERN TECHNIQUES ─────────────────── */

/* CSS Transitions */
.button {
  background: blue;
  transition: background 0.3s ease, transform 0.2s;
}
.button:hover {
  background: darkblue;
  transform: translateY(-3px);  /* move up on hover */
}

/* CSS Animations */
@keyframes bounce {
  0%   { transform: translateY(0); }

  100% { transform: translateY(0); }
}
.ball { animation: bounce 1s infinite ease-in-out; }

JavaScript · Topic 1 of 8

JavaScript Syntax & Data Types

JavaScript is the programming language of the web. It makes webpages interactive — clicking, moving, calculating, talking to servers!

🪄

What is JavaScript? HTML = structure, CSS = style, JavaScript = behavior. When you click a button and something happens, that's JavaScript! When a form validates, when content loads dynamically — JavaScript!

📊 Data Types — 7 Types of Values

TypeWhat it storesExamples

Variables & Scope

Variables are like boxes with labels — they store data. Scope decides where in your code you can access those boxes!

📦 Three Ways to Declare Variables

var vs let vs const
// ── CONST — cannot be reassigned (USE THIS by default!) ──
const PI = 3.14159;
const name = "Rahul";
const colors = ["red", "blue"];  // array CAN be modified
// PI = 5;  ← ERROR! Cannot reassign const

// ── LET — can be reassigned (USE when value changes) ──
let score = 0;
score = score + 10;  // ✅ OK to change
let userLoggedIn = false;
userLoggedIn = true;  // ✅ OK

// ── VAR — old way, AVOID in modern code ─────────────
var oldStyle = "avoid this";  // has weird hoisting behavior

/* RULE OF THUMB:
   Use const by default.
   Use let when you need to change the value.
   Never use var (it's old and buggy). */

🔭 Scope — Where Can You See the Variable?

GLOBAL SCOPE const appName = "WebLearn"; let totalUsers = 0;
FUNCTION SCOPE const discount = 0.1; let price = 100;
BLOCK SCOPE (if/for) let temp = price * discount;

✅ Can see: temp, discount, price, appName, totalUsers

✅ Can see: discount, price, appName, totalUsers   ❌ Cannot see: temp

✅ Can see: appName, totalUsers   ❌ Cannot see: discount, price, temp

Scope Examples
// GLOBAL — accessible everywhere
const appName = "WebLearn";

function calculate() {
  // FUNCTION SCOPE — only inside this function
  let price = 100;

  if (price > 50) {
    // BLOCK SCOPE — only inside this if block
    let discount = 10;
    const finalPrice = price - discount;
    console.log(finalPrice);  // ✅ 90
  }

  // console.log(discount);  ← ❌ ERROR! discount not here
  console.log(price);     // ✅ 100 (let stays in function)
  console.log(appName);   // ✅ "WebLearn" (global!)
}

// console.log(price);  ← ❌ ERROR! price only in function

// ── HOISTING (weird var behavior — another reason to avoid var!) ──
console.log(x);  // undefined (var is hoisted)
var x = 5;

// console.log(y);  ← ❌ ERROR! (let is NOT hoisted)
let y = 5;

Operators & Expressions

Operators are the tools to do math, compare values, and make logical decisions!

All Operators
// ── ARITHMETIC (math) ─────────────────────────
5 + 3    // 8  — addition
10 - 4   // 6  — subtraction
3 * 4    // 12 — multiplication
10 / 2   // 5  — division
10 % 3   // 1  — remainder (modulo)
2 ** 3   // 8  — power (2 to the 3rd)

let x = 5;
x++; // x = 6 (increment by 1)
x--; // x = 5 (decrement by 1)
x += 3; // x = 8  (same as x = x + 3)
x *= 2; // x = 16 (same as x = x * 2)

// ── COMPARISON (returns true or false) ────────
5 == "5"   // true  (equals in VALUE only — avoid!)
5 === "5"  // false (equals in VALUE and TYPE — use this!)
5 !== 3    // true  (not equal)
10 > 5    // true  (greater than)
3 >= 3    // true  (greater or equal)
2 < 5     // true  (less than)

// ── LOGICAL ───────────────────────────────────
true && true   // true  (AND — both must be true)
true && false  // false
true || false  // true  (OR — at least one must be true)
false || false // false
!true          // false (NOT — flips it)
!false         // true

// ── TERNARY (shorthand if/else) ───────────────
const age = 20;
const status = age >= 18 ? "adult" : "minor";
// condition ? valueIfTrue : valueIfFalse
// status = "adult"

// ── NULLISH COALESCING (??) ───────────────────
const username = null ?? "Guest";  // "Guest"
const title = "Rahul" ?? "Guest";  // "Rahul"

// ── STRING CONCATENATION ──────────────────────
"Hello" + " " + "World"  // "Hello World"
"Score: " + 100          // "Score: 100"

Control Flow Statements

Control flow means making decisions and repeating actions. JavaScript can think and loop — just like you!

If / Else / Switch
// ── IF / ELSE IF / ELSE ───────────────────────
const score = 75;

if (score >= 90) {
  console.log("Grade: A ⭐");
} else if (score >= 80) {
  console.log("Grade: B 👍");
} else if (score >= 70) {
  console.log("Grade: C 😊");  // ← This runs!
} else {
  console.log("Grade: F 😢");
}

// ── SWITCH ────────────────────────────────────
const day = "Monday";

switch(day) {
  case "Monday":
    console.log("Start of week 😴");
    break;
  case "Friday":
    console.log("Weekend soon! 🎉");
    break;
  default:
    console.log("Regular day");
}
Loops — Repeat Actions
// ── FOR LOOP ─────────────────────────────────
for (let i = 0; i < 5; i++) {
  console.log(`Counting: ${i}`);  // 0, 1, 2, 3, 4
}
// for(start; condition; increment)

// ── WHILE LOOP ────────────────────────────────
let count = 0;
while (count < 3) {
  console.log("count is: " + count);  // 0, 1, 2
  count++;
}

// ── FOR...OF — loop through arrays ────────────
const fruits = ["🍎 Apple", "🍌 Banana", "🥭 Mango"];

for (const fruit of fruits) {
  console.log(fruit);  // prints each fruit
}

// ── FOR...IN — loop through object keys ───────
const person = { name: "Rahul", age: 20, city: "Delhi" };

for (const key in person) {
  console.log(`${key}: ${person[key]}`);
  // name: Rahul, age: 20, city: Delhi
}

// ── BREAK & CONTINUE ─────────────────────────
for (let i = 0; i < 10; i++) {
  if (i === 3) continue;  // skip 3
  if (i === 6) break;     // stop at 6
  console.log(i);  // 0, 1, 2, 4, 5
}

JavaScript · Topic 5 of 8

Functions & Arrow Functions

Functions are reusable blocks of code. Write it once, use it many times. They're the heart of JavaScript!

🔧

Think of a function like a recipe: You write the recipe once (define the function). Every time you want to cook that dish, you follow the recipe (call the function). You can make it for different people (parameters)!

Functions — 3 Ways
// ── 1. FUNCTION DECLARATION ────────────────────
function greet(name) {
  return `Hello, ${name}! 👋`;
}
console.log(greet("Rahul"));   // "Hello, Rahul! 👋"
console.log(greet("Priya"));   // "Hello, Priya! 👋"

// ── 2. FUNCTION EXPRESSION ─────────────────────
const add = function(a, b) {
  return a + b;
};
console.log(add(5, 3));   // 8

// ── 3. ARROW FUNCTION (modern, shorter!) ───────
const multiply = (a, b) => {
  return a * b;
};

// Even shorter (one expression, no return needed)
const square   = n => n * n;         // one param, no ()
const getHello = () => "Hello!";   // no params, use ()

console.log(square(5));    // 25
console.log(getHello());   // "Hello!"

// ── DEFAULT PARAMETERS ─────────────────────────
function welcome(name = "Guest", emoji = "👋") {
  return `Welcome, ${name} ${emoji}!`;
}
console.log(welcome());          // "Welcome, Guest 👋!"
console.log(welcome("Rahul"));  // "Welcome, Rahul 👋!"

// ── REST PARAMETERS (any number of args) ───────
function sum(...numbers) {
  return numbers.reduce((total, n) => total + n, 0);
}
console.log(sum(1, 2, 3, 4, 5));  // 15

// ── HIGHER ORDER FUNCTIONS ─────────────────────
const nums = [1, 2, 3, 4, 5];

nums.forEach(n => console.log(n));           // print each
const doubled  = nums.map(n => n * 2);      // [2,4,6,8,10]
const evens    = nums.filter(n => n % 2 === 0); // [2,4]
const total    = nums.reduce((s, n) => s + n, 0); // 15

Arrays start counting from index 0 (not 1)! The first item is always at position 0.

Arrays
// ── CREATE an array ───────────────────────────
const fruits = ["Apple", "Banana", "Mango"];
const nums   = [1, 2, 3, 4, 5];
const mixed  = ["hello", 42, true, null]; // any types!

// ── ACCESS items (index starts at 0!) ─────────
fruits[0]  // "Apple" — first item
fruits[2]  // "Mango" — third item
fruits[fruits.length - 1]  // "Mango" — last item

// ── MODIFY ────────────────────────────────────
fruits.push("Grape");        // add to END → ["Apple","Banana","Mango","Grape"]
fruits.unshift("Cherry");   // add to START
fruits.pop();               // remove last item
fruits.shift();             // remove first item
fruits.splice(1, 1);        // remove 1 item at index 1

// ── USEFUL METHODS ────────────────────────────
fruits.length               // 3 — how many items
fruits.includes("Apple")    // true — is it in here?
fruits.indexOf("Banana")   // 1 — what index?
fruits.join(", ")          // "Apple, Banana, Mango"
fruits.reverse()           // reverse the array
fruits.sort()              // sort alphabetically
fruits.slice(0, 2)         // ["Apple","Banana"] (copy part)

// ── ARRAY SPREAD ──────────────────────────────
const more = ["Kiwi", "Peach"];
const all  = [...fruits, ...more];  // combine arrays

// ── DESTRUCTURING ─────────────────────────────
const [first, second, ...rest] = fruits;
// first = "Apple", second = "Banana", rest = [...]

Objects

const person = {

const { name, age, city = "Unknown" } = person;

// ── OBJECT METHODS ────────────────────────────
Object.keys(person)    // ["name","age","city",...]
Object.values(person)  // ["Rahul", 20, "Delhi",...]
Object.entries(person) // [["name","Rahul"],["age",20],...]

// ── SPREAD OPERATOR ───────────────────────────

Basic Event Handling

Events are things that happen on a webpage — clicks, hovers, typing, scrolling. JavaScript listens for them and reacts!

👂

Think of it like a waiter: The waiter (JavaScript) listens (addEventListener) for customers to snap their fingers (event). When they do, the waiter reacts (callback function runs)!

🖱️ Try These Events!

🎧 Listening for events... try clicking or typing!
Event Handling
// ── METHOD 1: HTML attribute (simple, quick) ──
// <button onclick="sayHello()">Click</button>

// ── METHOD 2: addEventListener (BEST WAY) ─────
const btn = document.querySelector("#myButton");

btn.addEventListener("click", function(event) {
  console.log("Button was clicked!");
  console.log(event.target);  // the element clicked
});

// Arrow function version
btn.addEventListener("click", (e) => {
  e.target.style.background = "red";
});

// ── COMMON EVENTS ─────────────────────────────
"click"       // mouse click
"dblclick"    // double click
"mouseover"   // mouse enters element
"mouseout"    // mouse leaves element
"keydown"     // key pressed down
"keyup"       // key released
"input"       // text typed in input
"change"      // dropdown/checkbox changed
"submit"      // form submitted
"load"        // page finished loading
"resize"      // browser window resized
"scroll"      // user scrolled

// ── FORM SUBMIT event ─────────────────────────
const form = document.querySelector("form");

form.addEventListener("submit", (e) => {
  e.preventDefault();  // STOP the page from reloading!
  const name = document.querySelector("#name").value;
  console.log(`Form submitted by: ${name}`);
});

// ── KEYBOARD events ───────────────────────────
document.addEventListener("keydown", (e) => {
  console.log(`Key pressed: ${e.key}`);
  if (e.key === "Enter") console.log("Enter pressed!");
  if (e.ctrlKey && e.key === "s") {
    e.preventDefault();
    console.log("Save triggered!");
  }
});

Introduction to Browser Interaction

JavaScript can read and change the webpage in real time using the DOM — the Document Object Model!

🌳

What is the DOM? The browser turns your HTML into a tree of objects (like a family tree). JavaScript can find any element in that tree and change it — text, style, add new elements, remove elements!

DOM Manipulation
// ── FINDING elements ──────────────────────────
document.getElementById("myId")       // find by ID
document.querySelector(".myClass")   // find FIRST match (like CSS)
document.querySelectorAll("p")       // find ALL matches (NodeList)
document.querySelector("nav a")     // CSS selectors work!

// ── READING content ───────────────────────────
const title = document.querySelector("h1");
title.textContent     // get/set text (no HTML tags)
title.innerHTML       // get/set including HTML tags
title.value           // for <input> elements

// ── CHANGING content ──────────────────────────
title.textContent = "New Title!";
title.innerHTML = "Hello <strong>World</strong>";

// ── CHANGING styles ───────────────────────────
title.style.color = "red";
title.style.fontSize = "2rem";
title.style.display = "none";        // hide it!

// ── CLASSES ───────────────────────────────────
title.classList.add("active");      // add class
title.classList.remove("active");   // remove class
title.classList.toggle("dark");    // add if absent, remove if present
title.classList.contains("active"); // true or false?

// ── ATTRIBUTES ────────────────────────────────
const img = document.querySelector("img");
img.setAttribute("src", "new-photo.jpg");
img.getAttribute("alt");

// ── CREATE new elements ───────────────────────
const newCard = document.createElement("div");
newCard.className = "card";
newCard.textContent = "I am a new card!";
document.body.appendChild(newCard);   // add to page

// ── REMOVE elements ───────────────────────────
newCard.remove();   // remove from page

// ── BROWSER DIALOGS ───────────────────────────
alert("Hello!");                           // popup message
const name = prompt("What is your name?");   // ask for input
const ok   = confirm("Are you sure?");       // yes/no dialog

// ── CONSOLE METHODS ───────────────────────────
console.log("Normal message");
console.warn("Warning!");
console.error("Error!");
console.table([{ name: "Rahul", age: 20 }]);  // nice table!

Git saves a complete history of every change you make to your code. GitHub hosts it online so you can collaborate with others!

🌿 What is a Branch?

Like a copy of your project where you can experiment freely. When it works, you merge it back to main. Your original code is always safe!

⚔️ Merge Conflicts

Happen when two people edit the same line. Git marks the conflict with <<< and >>> symbols. You manually pick which version to keep.

💡 Good Commit Messages

❌ "changes"
❌ "stuff"

🔑 Golden Rule

NEVER commit your passwords, API keys, or .env files! Once pushed to GitHub (even private), they can be exposed. Use .gitignore!
